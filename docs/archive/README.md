## When to Reference Archived Docs

These documents may be useful for:
- Understanding the evolution of architectural decisions

- Training materials showing problem-solving process

---

## Archival Policy

**Archive when:**
1. Superseded by more current documentation
2. Describes problems that have been fully resolved
3. Contains planning for work that has been completed
4. Is primarily of historical interest

**Keep active when:**
1. Serves as current architectural reference
2. Documents ongoing or future work
3. Provides operational guidance
4. Is frequently referenced by the team
